# sendspin-docker — Agent Context

## Project

`sendspin-docker` is an open-source Docker image that turns a USB speaker into a permanent
[Sendspin](https://github.com/Sendspin-Protocol/sendspin) receiver. It runs `sendspin daemon`,
talks directly to ALSA (bypassing PulseAudio/PipeWire), and advertises itself via mDNS on the
local network. Published to `ghcr.io/<owner>/sendspin-docker`.

## Architecture Decisions

These decisions were made deliberately — do not revisit without strong justification.

| Decision | Choice | Reason |
|---|---|---|
| Base image | `python:3.12-slim` | Alpine fails: sounddevice ships manylinux wheels (glibc), incompatible with musl libc |
| User | Non-root `sendspin` (UID 1000) | Security best practice for distributed open-source images |
| Audio access | `devices: /dev/snd` + `group_add: "${AUDIO_GID:-29}"` | `devices` alone is not enough for non-root; `privileged: true` is a security red flag |
| Audio GID | Default `29` | Standard on Debian, Ubuntu, TrueNAS; overridable via `AUDIO_GID` env var |
| Network | `network_mode: host` | Required for mDNS advertisement and discovery (`_sendspin._tcp.local.`) |
| No PulseAudio | Intentional | Container speaks ALSA directly; host audio server would conflict |
| Config volume | `/home/sendspin/.config/sendspin` | Persists sendspin settings across restarts |
| Client ID | Auto-derived from `SENDSPIN_NAME` | Stable across reboots without manual config; slugified (e.g. `living-room`) |
| Device auto-detect | Parse `sendspin --list-audio-devices` | Skips virtual devices (default, sysdefault, dmix, null); takes first real device |

## Environment Variables

All variables are optional. The container works out of the box with a single USB speaker.

| Variable | Default | Description |
|---|---|---|
| `SENDSPIN_NAME` | `Sendspin Speaker` | Friendly name shown in the Sendspin app |
| `SENDSPIN_AUDIO_DEVICE` | *(auto-detected)* | Device index or ALSA name. Auto-detects first real device if empty |
| `SENDSPIN_PORT` | `8927` | Listening port for incoming server connections |
| `SENDSPIN_HARDWARE_VOLUME` | `true` | Enable hardware/system volume control |
| `SENDSPIN_SERVER_URL` | *(mDNS)* | WebSocket URL of the server. Leave empty for auto-discovery |
| `SENDSPIN_AUDIO_FORMAT` | *(auto)* | Preferred format e.g. `flac:48000:24:2` |
| `SENDSPIN_STATIC_DELAY_MS` | | Latency compensation in ms (typically negative) |
| `SENDSPIN_CLIENT_ID` | *(derived from name)* | Unique client ID. Auto-derived from `SENDSPIN_NAME` if not set |
| `SENDSPIN_HOOK_START` | | Command to run when audio stream starts |
| `SENDSPIN_HOOK_STOP` | | Command to run when audio stream stops |
| `SENDSPIN_HOOK_SET_VOLUME` | | External volume control script (receives 0–100 as last argument) |
| `SENDSPIN_LOG_LEVEL` | `INFO` | Log level: `DEBUG` `INFO` `WARNING` `ERROR` `CRITICAL` |
| `AUDIO_GID` | `29` | Audio group GID on the host. Check with `stat -c '%g' /dev/snd/controlC0` |

## Build & Test

```bash
# Build the image locally
docker compose build --no-cache

# List available audio devices inside the container
docker compose run --rm sendspin list-devices

# Start the receiver (foreground, see logs)
docker compose up

# Start in background
docker compose up -d

# Follow logs
docker compose logs -f

# Rebuild and restart after changes
docker compose up -d --build
```

The USB speaker must be plugged in before starting the container. If unplugged and replugged, restart the container.

## Commit Conventions & Release-Please

This project uses [Conventional Commits](https://www.conventionalcommits.org/) and
[release-please](https://github.com/googleapis/release-please) for automated versioning.

### How it works

1. You push conventional commits to `main`
2. release-please opens a PR titled **"chore: release X.Y.Z"** with an updated `CHANGELOG.md` and `version.txt`
3. You review and **merge the PR** (human-in-the-loop — releases are never automatic)
4. release-please creates a git tag `vX.Y.Z`
5. `docker-publish.yml` detects the tag and pushes the image to ghcr.io with full semver tags

### Commit format

```
<type>(<optional scope>): <description>

[optional body]

[optional footer: BREAKING CHANGE: <description>]
```

### Version bump rules

| Commit type | Version bump | Example |
|---|---|---|
| `fix:` | patch `1.0.x` | `fix: handle missing audio device gracefully` |
| `feat:` | minor `1.x.0` | `feat: add support for SENDSPIN_HOOK_SET_VOLUME` |
| `feat!:` or `BREAKING CHANGE:` footer | major `x.0.0` | `feat!: rename SENDSPIN_DEVICE to SENDSPIN_AUDIO_DEVICE` |
| `chore:` `docs:` `ci:` `refactor:` `test:` | no release | `docs: update README with multi-speaker example` |

### Image tags produced

| Trigger | Tags |
|---|---|
| Push to `main` | `latest` |
| Git tag `v1.2.3` | `1.2.3` + `1.2` + `1` + `latest` |
| Pull request | Build only, no push |

## Known Constraints

| What | Why it doesn't work |
|---|---|
| `python:3.12-alpine` | sounddevice ships manylinux (glibc) wheels — the `.so` extension fails to load on musl libc. Compiling from source produces a 308MB image, larger than `slim` (378MB), and still fails due to `ctypes.util.find_library` musl limitations |
| `privileged: true` | Unnecessary and flagged by security scanners (Trivy, Snyk, Docker Scout) |
| Bridge networking | mDNS multicast requires `network_mode: host` — bridge mode breaks service discovery |
| `disallow-other-stacks=yes` in Avahi | Prevents the container from binding to port 5353 for mDNS. Workaround: set `SENDSPIN_SERVER_URL` to connect directly, bypassing mDNS |
| `manufacturer` field in Sendspin UI | Hardcoded to `None` in `sendspin/utils.py` — not configurable without a PR to the sendspin repo |
| Hot-unplug of USB speaker | Container must be restarted after USB reconnect |
